### 适配器模式
- 适配器模式将某个接口转化成客户端期望的另一个接口
- 主要目的是兼容性
- 分类
    - 类适配器模式
        - 有一定局限性，需要继承一个src类
        - 增加使用成本，类的方法会暴露
        - 灵活性增强，可以根据需求重写父类方法
    - 对象适配器模式
        - 和对象适配器适用同一种思想
        - 解决了类适配器的局限性问题
        - 使用成本低，更灵活
    - 接口适配器模式
        - 有选择的实现接口
        - 设计一个抽象类，然后去实现一个接口，最后选择实现相应的方法或者重写
