### 简单工厂模式
- 简单工厂模式是由一个对象决定创建出哪一种产品类的实例
- 定义一个创建对象的类，由这个类来封装实例化对象的行为
- 适用：我们会用到大量的创建某种、某类或者某批对象时，就会使用工厂模式。

### 工厂模式
- 工厂方法模式：在静态工厂模式上进行改进。
- 工厂类被抽象化，用于实例化特定产品的代码被转移到实现抽象方法的子类中

### 抽象工厂模式
- 抽象工厂模式：定义了一个接口用于创建相关或有依赖的对象，而无需指明具体类
- 可以看作将简单抽象工厂模式和工厂方法模式进行整合
 
### 工厂模式的意义
- 将实例化对象的代码提取出来，放到一个类中统一维护和管理，达到和主项目的依赖关系的解耦。从而提高项目的扩展性和维护性
